==============================
一、修复当前仓库列表数据同步问题
==============================

解决以下问题：

### 1. 删除仓库后列表状态同步

当前问题：

删除仓库后：

- 仓库数量显示不正确。
- 排序编号错乱。
- 表格行号和内部仓库列表不同步。
- 后续点击更新按钮可能更新错误仓库。


要求：

删除仓库后：

1. 同步更新：

- self.repos
- 表格行
- 仓库缓存
- 相关计数
- 排序状态


2. 删除后重新校准：

- row索引
- 仓库路径映射
- 按钮绑定关系


禁止：

通过固定row保存仓库逻辑。


建议：

所有按钮和操作必须通过：

repo_path

作为唯一标识。

不要依赖：

row index。


---

### 2. 更新按钮错乱修复

当前问题：

点击某个仓库更新：

可能因为删除其他仓库后：

- row发生变化
- lambda绑定旧row
- 更新错误仓库


修改：

更新按钮绑定：

不要：

lambda row


改为：

绑定：

repo_path


例如：

update_repo(repo_path)


点击按钮时：

通过路径查找当前行。


要求：

删除、更新、右键菜单全部使用：

repo_path

作为唯一ID。
